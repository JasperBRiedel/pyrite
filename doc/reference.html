run() - Configure and start the engine

Initialise the engine with the passed configuration structure

Configuration structure example....
pyrite.run config config Engine configuration dictionary

Configuration structure

{
"application_name": application_name
"application_version": application_version
"viewport_scale": viewport_scale
"viewport_width": viewport_width
"viewport_height": viewport_height
"tileset_path": tileset_path
"tileset_width": tileset_width
"tileset_height": tileset_height
"tile_names": tile_names
} application_name Used to identify the application and set the window title.
application_version Should be a version number complying to semantic versioning.
viewport_scale Scale factor provided as a positive integer.
viewport_width Initial width of the viewport in tiles.
viewport_height Initial height of the viewport in tiles.
tileset_path Name of the tileset file, including the extension.
tileset_width Horizontal tile count in the tileset.
tileset_height Vertical tile count in the tileset.
tile_names An array of tile names to be assigned to tiles in left to right, top to bottom order. Tiles without pixel data won't be indexed.

__entry__() - Engine life cycle callback

The entry callback is invoked by the engine after initialisation. pyrite.run, pyrite.timestep, and pyrite.poll_events must be invoked inside this function.

Must be defined inside the entry file / module
__entry__():
pass

timestep() - Create update loops at different intervals

Allows the creation of various update loops that run at a predetermined interval
pyrite.timestep interval interval Update rate in hertz. ie. A value of 10 will update 10 times per second.

exit() - Initiate a graceful exit of the engine

Instruct the engine to gracefully exit.
pyrite.exit

mouse_position() - Get the location of the mouse

Get the location of the mouse in tile coordinates.
pyrite.mouse_position return x y x Position on the X axis in tile coordinates
y Position on the Y axis in tile coordinates

set_viewport() - Configure the dimensions and scale of the window

Set the width and height of the window in tile increments.

Set the tile scale factor.
pyrite.set_viewport width height scale width Width in tiles of the window. Must be a whole number.
height Height in tiles of the window. Must be a whole number.
scale Scale factor of the tiles. Must be a whole number.

set_tile() - Set top layer tile properties

Set the display properties of the top layer tile in the scene.
pyrite.set_tile x y name red green blue flip_x flip_y (x, y) The x and y coordinate tuple of the tile to be set.
name The name of the tile sprite as defined in the configuration structure passed to pyrite.run(...)
(red, green, blue) The RGB colour tuple, multiplies the tile colours by the modifier values, allowing colour shifting and colouring of grey scale sprites.
(flip_x, flip_y) Tile sprite flip tuple, boolean value determines if the tile should be flipped on that axis.

set_tile() - Set top and bottom layer tile properties

Set the display properties of the top and bottom layer tiles in the scene.
pyrite.set_tile x y
top_name top_red top_green top_blue top_flip_x top_flip_y
bottom_name bottom_red bottom_green bottom_blue bottom_flip_x bottom_flip_y
(x, y) The x and y coordinate tuple of the tile to be set.
top_name The name of the tile sprite as defined in the configuration structure passed to pyrite.run(...)
(top_red, top_green, top_blue) The RGB colour tuple, multiplies the tile colours by the modifier values, allowing colour shifting and colouring of grey scale sprites.
(top_flip_x, top_flip_y) Tile sprite flip tuple, boolean value determines if the tile should be flipped on that axis.
bottom_name The name of the tile sprite as defined in the configuration structure passed to pyrite.run(...)
(bottom_red, bottom_green, bottom_blue) The RGB colour tuple, multiplies the tile colours by the modifier values, allowing colour shifting and colouring of grey scale sprites.
(bottom_flip_x, bottom_flip_y) Tile sprite flip tuple, boolean value determines if the tile should be flipped on that axis.

button_down() - Get the down state of a button input

Returns true if the specified button is down.
pyrite.button_down button return down button String button constant.
down True if the specified button is currently down.

poll_events() - Responding to engine events

Returns a list of events that's usually iterated and drives some amount of game logic.

Use events to respond to button presses, mouse scroll, and text entry.

Event data access information here....
pyrite.poll_events return event_queue event_queue An array of events popped of the engine event queue.

resource_read() - Loading packaged resources

Read a packaged resource into a string and return it.
pyrite.resource_read name return data name Name of a packaged file including the file extensions.
data File data as a string.

resource_exists() - Check if packaged resources exist

Check a packaged resource to see if it exists.
pyrite.resource_exists name return exists name Name of a packaged file including the file extensions.
exists Boolean true if the file exists.

Event constants

button
scroll
text

Button event

{ "type": "button", "button": button_constant "transition": transition_constant } button_constant String value containing a button constant.
transition_constant String value containing a transition constant.

Transition constants

pressed
released

Scroll event

{ "type": "scroll", "x": x_delta "y": y_delta } x_delta A signed integer containing the scroll delta since the last event.
y_delta A signed integer containing the scroll delta since the last event.

Text event

{ "type": "text", "text": text_input } text_input A string containing any characters entered since the last event.
